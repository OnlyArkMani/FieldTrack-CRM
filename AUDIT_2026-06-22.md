## Part A — New features delivered this pass

### Change 1 — Geofence team assignment ✅
- **Migration** `0004_geofence_team_assignment.py`: adds `geofences.team_id` (BigInteger FK → `teams.id`, `ON DELETE SET NULL`, nullable = universal) and `geofences.scope VARCHAR(10) NOT NULL DEFAULT 'UNIVERSAL'`, plus index `idx_geofences_team_id`. Existing rows safely become UNIVERSAL — no data loss. Chain is linear `0001→0002→0003→0004`.
- **Model** `Geofence`: `scope` + `team_id` columns + index.
- **Schema** `GeofenceCreate`/`GeofenceUpdate`: `scope`/`team_id` with validators — TEAM requires `team_id` (raises `ValueError`), UNIVERSAL silently clears `team_id`. `GeofenceOut` now carries `scope`, `team_id`, `team_name`.
- **Repository**: `get_geofences_for_user(user_id, team_id)` (universal + own-team, active), `get_all_geofences_admin()` (all, with `team_name` join), and `geofences_containing_for_team(lat,lng,team_id)` for scope-aware point-in-polygon.
- **Service**: create/update persist scope+team (team existence validated), `list_for_user`, `list_all_admin`; `check_geofence_events` resolves the employee's `team_id` and only evaluates relevant zones — universal fire for everyone, team zones only for their team.
- **API** `GET /geofences`: ADMIN → all (with `team_name`); SUPERVISOR/EMPLOYEE → team + universal.
- **Admin web**: create modal has a **Visibility** section (Universal / Team radio-cards, amber border on select, team dropdown with member counts); list shows a per-row **Scope** badge (globe/Universal grey, people/team-name amber) and a filter dropdown (All / Universal / by team).

### Change 2 — Employee & supervisor map show assigned geofences ✅
- `geofence_layer.dart`: Circle + Polygon rendering (amber 15% fill, 2px border) **plus centered name labels** (amber pill, white ellipsised text).
- `map_screen.dart`: **Show/Hide Zones** toggle (top-right, eye icon, `AnimatedSwitcher`, default visible) on both employee and supervisor maps; zones gated by the toggle.
- **Tap → bottom sheet** via point-in-polygon/circle hit-testing (`Geofence.contains`):
  - Employee: zone name, scope, shape + area/radius, and "You visited today: Xh Ym" (from `/geofences/employee/{id}/today`) or "Not visited yet".
  - Supervisor: zone name + scope, per-member dwell list, and "X of Y team members visited today" (from `/geofences/{id}/presence`).
- New providers (`showZonesProvider`, `employeeZonesTodayProvider`, `zonePresenceProvider`) + repo methods (`employeeZonesToday`, `zonePresence`) + models (`ZoneVisit`, `ZonePresence`).

### Change 3 — Employee self-notification on enter/exit ✅
- `geofence_service._notify` now notifies **both** supervisor (unchanged wording) **and** the employee:
  - ENTER → "Zone Entry" / "You entered {zone}" / type `GEOFENCE_ENTER`.
  - EXIT → "Zone Exit" / "You left {zone} after N minutes" (duration = exit − most-recent ENTER for that zone, rounded) / type `GEOFENCE_EXIT`.
- Both pushes are fire-and-forget (`_push` swallows all FCM errors), in-app rows remain the source of truth.
- Flutter: `GEOFENCE_ENTER`/`GEOFENCE_EXIT` icons updated to location-pin variants (`where_to_vote` / `wrong_location`) and given green/coral accents in the list.

### Change 4 — Geofence compliance report ✅
- New `ReportType.GEOFENCE_COMPLIANCE`. Authorize branch makes **team_id mandatory** (400 if missing), employees forbidden, supervisors must supervise the team, admin any.
- `_geofence_compliance_data`: per employee × day × assigned zone → `visited`, first entry, last exit, dwell minutes, visit count; a per-employee summary row (zones assigned/visited, compliance %). Assigned zones = universal + team (`assigned_geofences_for_team`).
- **Excel conditional formatting** via new `ReportTable.row_styles`: visited rows light-green, not-visited light-coral, summary rows amber/bold. CSV supported; **PDF rejected (400)** like the distance report.
- **Admin web**: "Geofence Compliance" in the dropdown; team selector becomes **required** (red asterisk, Generate disabled until chosen); description shown; PDF hidden.
- **Flutter (supervisor only)**: "Compliance" chip; team selector pre-selected to their team and **locked**; PDF hidden; same generate/poll/download flow.

## Part B — APK production readiness audit (Change 5)

| Item | Finding | Action |
|---|---|---|
| `minSdk >= 21` | Was `flutter.minSdkVersion` (implicit). | Pinned `minSdk = maxOf(21, flutter.minSdkVersion)` — never below the floor. |
| `targetSdk = 34` | Was `flutter.targetSdkVersion` (could drift). | Pinned `targetSdk = 34`. |
| `versionCode`/`versionName` | Driven by pubspec `version: 0.1.0+1` (set, not 1/1.0). | Documented: bump pubspec build number before each store upload. |
| Release signing | Signed with **debug** keys. | Added a `key.properties`-driven release `signingConfig`; falls back to debug for internal testing; documented "create keystore before Play submission". |
| `shrinkResources` / `minifyEnabled` | Both already `true` for release. | ✅ unchanged. |
| ProGuard — Flutter/Dio/Firebase/sqflite/geolocator | Present. | ✅ |
| **ProGuard — background_locator_2** | **MISSING — critical.** The `@pragma('vm:entry-point')` callback dispatcher (`yukams.app.background_locator_2.*`) was not kept; R8 would strip/rename it and **background GPS silently dies in release only.** | **Added** keep rules for `yukams.app.background_locator_2.**`, `yukams.app.**`, `@androidx.annotation.Keep`, and `flutter_local_notifications` (`com.dexterous.**`). |
| `env.dart` API URL | Reads `API_BASE_URL` from `flutter_dotenv`; **no localhost/dev-IP fallback**, throws if missing. | ✅ no change needed. |
| Debug flags in release | `debugPrint` calls are exception-logging only (no PII/credentials); no hardcoded test creds; no asserts in hot paths. | ✅ acceptable; `debugPrint` is a no-op concern only, not a leak. |
| `AndroidManifest` permissions | FINE/COARSE/BACKGROUND location, FOREGROUND_SERVICE(+LOCATION), BOOT_COMPLETED, INTERNET, WAKE_LOCK, POST_NOTIFICATIONS all present. | ✅ |
| `foregroundServiceType="location"` | Present on `IsolateHolderService`. | ✅ (required for Android 14). |
| **`allowBackup="false"`** | **MISSING.** | **Added** `allowBackup="false"`, `fullBackupContent="false"`, and `dataExtractionRules` (excludes root/file/db/sharedpref from cloud backup + device transfer). |
| **`networkSecurityConfig`** | **MISSING.** | **Added** `@xml/network_security_config` (release: **cleartext disabled app-wide**, system trust anchors). A **debug override** (`src/debug/res/xml/...`) re-permits cleartext to localhost/LAN for `flutter run`. |

### Build verification checklist (manual — run on a real device)

```
[ ] APK installs without "Parse Error"
[ ] Login screen loads (confirms API_BASE_URL is correct)
[ ] Login succeeds with real credentials
[ ] Attendance START works (confirms backend connection)
[ ] GPS dot appears on map within 3 minutes of START
[ ] Geofence zone visible on map (confirms /geofences scope API)
[ ] Lock screen + 10 minutes → GPS still tracking (foreground service survived → proves the new ProGuard keep rule works)
[ ] Airplane mode → attendance action → re-enable network → sync within 60s
[ ] Geofence entry → BOTH employee + supervisor notified within 30s
[ ] Logout → no screen reachable (JWT invalidated / refresh revoked)
```

> Two of these (the foreground-service survival check and the geofence-notification check) are the highest-value regression tests — the ProGuard fix and Change 3 respectively. ❓ These are device-only and cannot be verified statically.
